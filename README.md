# blogシステム (blog System)

本リポジトリは、自身のJava開発における技術スタック（Spring Boot / MyBatis / Thymeleaf）の即戦力性を証明、およびブランク期間中の技術キャッチアップのために一から単独で設計・構築した、フルスタックのWebアプリケーション（完全版）です。
画面開発からデータベース設計、セキュリティ実装まで、実務に即した標準的なアーキテクチャでクリーンコードを意識して作成しています。

---

## 🛠 技術スタック (Tech Stack)

### バックエンド (Backend)
* **Java** 11 （※あなたのバージョンに合わせて変更してください）
* **Spring Boot** 3.3.4
* **MyBatis** (Mapper XMLによる動的SQLの最適化)
* **Spring Security** (認証・認可、ロールベースのアクセス制御)

### フロントエンド (Frontend)
* **Thymeleaf** (サーバーサイド・テンプレートエンジンによるセキュアな画面レンダリング)
* **Bootstrap 4 / **JavaScript** (レスポンシブWebデザイン対応)

### データベース & インフラ (Database & Infrastructure)
* **MySQL 8.0** (インデックス最適化、トランザクション管理)
* **Maven** (依存関係管理)

---

## 🎯 主な機能 (Features)

* **データCRUD & 動的検索機能**: 
  MyBatis の動的SQL（`<if>`, `<where>`タグ等）を活用し、大量データに対してもパフォーマンスを維持した複合条件検索。
* **SQLによるネイティブページネーション**: 
  フレームワークのプラグインに依存せず、MySQL の `LIMIT` および `OFFSET` を用いたSQLレベルでの確実なページング処理。
* **カスタムエラーハンドリング**: 
  `ErrorViewResolver` インターフェースを実装（implements）し、システム内で発生する各種例外を捕捉。ユーザーに不適切なエラー画面を見せない、カスタマイズされたエラーページ遷移を構築。

---

## 🚀 実績・取り組み (Key Implementations & Optimizations)

* **MyBatisによるクエリの高速化と最適化**:
  複雑なテーブル結合（JOIN）が発生する業務ロジックにおいて、N+1問題を回避するために適切なフェッチ設定と適切なインデックス（Index）設計を実施。検索レスポンスの向上を実現しました。
* **Thymeleafを活用したセキュアな画面開発**:
  XSS（クロスサイトスクリプティング）等の脆弱性を防ぐため、Thymeleafの標準エスケープ出力を徹底。また、フラグメント（Fragment）化によるコンポーネントの共通化を行い、モジュールとしての保守性を大幅に高めました。
* **疎結合な3層アーキテクチャの徹底**:
  Controller - Service - Mapper(DAO) の3層構造を厳格に分離。各レイヤーの責務を明確にすることで、属人化を排除し、テストコード（JUnit）の導入や将来的な仕様変更に強い設計を意識しました。
